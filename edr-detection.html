Detection & Response Operations

Alert triage, investigation workflows, and response actions in Trellix EDR.

Alert Triage Workflow

Initial Assessment
Review alert severity, affected systems, and detection type. Check if related alerts exist.
Context Gathering
Examine process tree, file details, network connections. Determine scope of activity.
Verdict Decision
True Positive ‚Üí Escalate to incident. False Positive ‚Üí Document and tune detection.
Response Action
Contain threat if confirmed. Isolate host, kill process, collect evidence.

Investigation Tools

Timeline Analysis

Chronological view of all events on a system. Use to:

  • Identify initial compromise vector
  • Track lateral movement
  • Document attacker activity sequence
  • Establish timeline for incident report

Process Tree

Visual parent-child process relationships. Reveals:

  • Suspicious process spawning (e.g., Word spawning PowerShell)
  • Injection targets
  • Persistence mechanisms
  • Complete attack chain

Telemetry Search

Query raw telemetry across all endpoints:

Example Queries 
# Find PowerShell with encoded commands
process_name:powershell.exe AND command_line:*-enc*

# Find LSASS access
target_process:lsass.exe AND event_type:process_access

# Find scheduled task creation
event_type:registry_write AND registry_path:*TaskCache*

# Find outbound connections to suspicious ports
event_type:network_connection AND dest_port:(4444 OR 5555 OR 8080)

Response Actions

Action Impact Use Case
Network Isolate High - blocks all traffic Active breach containment
Kill Process Medium - terminates process Stop malicious execution
Quarantine File Low - moves file Remove malware from disk
Collect File None - forensic only Retrieve samples for analysis
Memory Dump None - forensic only Capture process memory
Remote Shell Variable Manual investigation/remediation

Network Isolation

Network isolation is highly disruptive. Have a release plan ready. The isolated host can still communicate with EDR cloud for commands.

Automated Response Playbooks

Example: Ransomware Response

  1. Trigger: Ransomware detection with high confidence
  2. Auto-Action: Network isolate affected host
  3. Auto-Action: Kill suspicious process
  4. Notification: Alert SOC and IT
  5. Manual Step: Analyst reviews and determines scope
  6. Manual Step: Remediate and restore
