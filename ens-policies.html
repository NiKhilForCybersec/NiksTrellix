On-Access Scan Configuration

Scan Settings:
  • When to scan: On read and write
  • Network drives: Disable (use network DLP instead)
  • Archives: Enable, depth 2-3
  • MIME: Enable
Actions:
  • Threat detected: Clean first, then Delete if clean fails
  • Unwanted programs: Block
  • Enable scan cache for clean files
  • Trust McAfee-signed processes
  • Set appropriate scan timeout (45 seconds default)
  • Use extension-based scanning for better performance

Exclusion Security

Exclusions create security gaps. Only exclude paths that cause documented issues. Prefer hash-based exclusions over path-based when possible.

Common Exclusions 
# SQL Server
C:\Program Files\Microsoft SQL Server\*\MSSQL\DATA\
*.mdf, *.ldf, *.ndf

# Exchange Server
C:\Program Files\Microsoft\Exchange Server\*
*.edb, *.stm, *.log

# IIS
C:\inetpub\logs\*
C:\Windows\System32\inetsrv\*

# Development
*\node_modules\*
*.pdb, *.obj

# Backup Software
VSS Writer exclusions per vendor docs
Exclusion Type Security Risk Use When
By Hash (SHA-256) Low Specific file false positive
By Certificate Low-Medium Trusted vendor software
By File Path Medium Application directories
By Extension High Avoid if possible

Tuning Strategy

Enable Exploit Prevention rules in "Report" mode first. Monitor for false positives for 1-2 weeks, create exclusions, then switch to "Block" mode.
