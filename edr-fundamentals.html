What is Trellix EDR?

Trellix EDR (Endpoint Detection and Response) provides continuous endpoint monitoring, behavioral detection, and automated response capabilities. It collects rich telemetry from endpoints to enable threat hunting, forensic investigation, and rapid incident response.

EPP vs EDR: Understanding the Difference

Endpoint Protection Platform (EPP) and EDR serve complementary but distinct purposes. Most organizations deploy both for comprehensive endpoint security.

Aspect EPP (ENS) EDR
Primary Purpose Prevention Detection & Response
Protection Model Block known threats Detect and investigate unknown threats
Data Collection Limited (threat events only) Extensive (all endpoint activity)
Detection Method Signatures, heuristics Behavioral analysis, ML, IOCs
Investigation Basic alert details Full timeline, process tree, forensics
Response Automatic (clean/delete) Manual + Automated playbooks
Trellix Product Endpoint Security (ENS) Trellix EDR

Deployment Recommendation

Deploy ENS for prevention and EDR for detection/response. The two products share the same agent framework and integrate seamlessly through ePO management.

Threat Hunting

EDR enables proactive threat hunting by providing searchable telemetry data and investigation tools.

Hunting Workflow

Hypothesis Development
Based on threat intelligence, recent incidents, or anomaly detection. Example: "Are there signs of Cobalt Strike beacons in our environment?"
Query Construction
Build queries to search telemetry for indicators. Search for known Cobalt Strike IOCs, process patterns, or network behaviors.
Investigation
Review results, pivot on findings, build process trees, analyze timelines. Determine if activity is malicious or benign.
Response
Contain threats if found. Document findings. Create detection rules to catch similar activity in the future.

Example Hunting Queries

process_name:powershell.exe AND 
command_line:(*-enc* OR *-encoded* OR *-e * OR *frombase64*)
