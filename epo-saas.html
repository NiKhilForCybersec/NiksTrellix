What is ePO SaaS?

Trellix ePO SaaS (formerly MVISION ePO) is a cloud-hosted version of ePolicy Orchestrator. Trellix manages the infrastructure while you focus on security policy and endpoint management.

Product Naming

ePO SaaS was previously known as MVISION ePO and McAfee ePO Cloud. All refer to the same cloud-hosted management platform.

Onboarding to ePO SaaS

Provision Tenant

Work with Trellix sales/support to provision your ePO SaaS tenant. You'll receive:

  • Tenant URL (e.g., company.manage.trellix.com)
  • Initial admin credentials
  • License entitlements
Configure Identity

Set up user authentication:

  • Configure SSO with your IdP (Azure AD, Okta, etc.)
  • Set up MFA requirements
  • Create admin roles and permissions
Deploy Agents

Generate agent installation package from ePO SaaS console:

  • Download FramePkg installer
  • Package includes cloud connection settings
  • Deploy via SCCM, Intune, GPO, or manual
Configure Policies

Set up security policies for your endpoints:

  • Import existing policies (if migrating)
  • Configure ENS policies
  • Set up DLP rules (if applicable)

SaaS Limitations

Understand these limitations before choosing ePO SaaS over on-premises deployment.

Cannot run custom SQL queries or access the database directly. Reporting limited to built-in reports and API exports.

Endpoints must have internet access to communicate with ePO SaaS. Air-gapped environments require on-premises ePO.

Not all Trellix products support ePO SaaS management. Network DLP and some legacy products require on-premises ePO.

Data stored in Trellix cloud infrastructure. Consider regulatory requirements (GDPR, data sovereignty) when choosing regions.
