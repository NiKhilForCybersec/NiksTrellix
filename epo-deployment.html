Installation Process

Prepare SQL Server

Create SQL login and database for ePO:

SQL Setup 
-- Create SQL Login
CREATE LOGIN ePOAdmin WITH PASSWORD = 'StrongPassword123!';

-- Create Database
CREATE DATABASE ePO_SERVERNAME;

-- Grant Permissions
USE ePO_SERVERNAME;
CREATE USER ePOAdmin FOR LOGIN ePOAdmin;
EXEC sp_addrolemember 'db_owner', 'ePOAdmin';
Run ePO Installer

Launch the ePO installation wizard and provide:

  • SQL Server hostname and instance
  • Database credentials
  • ePO admin account credentials
  • Certificate configuration (or use self-signed)
Configure Certificates

For production, import enterprise CA-signed certificates:

  • Console certificate (HTTPS)
  • Agent-server communication certificate
  • Agent handler certificates (if applicable)
Initial Configuration

Access console at https://epo-server:8443 and complete:

  • License activation
  • Repository configuration
  • Initial product check-in

Agent Deployment Methods

ePO Remote Push

Push agents directly from ePO console to discovered systems.

Requirements:
  • Admin$ share accessible
  • RPC ports open (TCP 135, 445)
  • Credentials with local admin rights
Process:
  1. Menu ‚Üí Systems ‚Üí New Systems
  2. Select "Push agents to target systems"
  3. Enter hostnames or IP range
  4. Provide admin credentials
  5. Select agent version and products

Manual Package Installation

Export agent installer for manual deployment.

Silent Install Command 
# Basic silent install
FramePkg.exe /install=agent /silent

# With logging
FramePkg.exe /install=agent /silent /log=C:\temp\agent.log

# Specify server
FramePkg.exe /install=agent /silent /server=epo.company.com

GPO / SCCM Deployment

Enterprise deployment via existing management tools.

GPO Startup Script:
deploy_agent.bat 
@echo off
REM Check if agent already installed
if exist "C:\Program Files\McAfee\Agent\cmdagent.exe" exit /b 0

REM Install from network share
\\fileserver\software\McAfee\FramePkg.exe /install=agent /silent

exit /b %ERRORLEVEL%

URL-Based Installation

Self-service installation via web URL.

  1. Enable URL deployment in ePO settings
  2. Distribute URL to users: https://epo-server:8443/Software/install
  3. Users download and run installer
  4. Agent auto-registers with ePO

BYOD/Remote Users

URL installation is ideal for remote workers and BYOD scenarios where direct network access isn't available.

Upgrade Procedures

Upgrade Planning

Always review release notes and test upgrades in a non-production environment first. Create full database backups before upgrading.

Pre-Upgrade Checklist
  • Review release notes and known issues
  • Verify hardware meets new requirements
  • Backup ePO database (full backup)
  • Backup ePO server configuration
  • Document current extension versions
  • Schedule maintenance window
Upgrade ePO Server
  • Stop all ePO services
  • Run upgrade installer
  • Follow wizard prompts
  • Verify services restart successfully
Upgrade Agent Handlers

Upgrade agent handlers after ePO server. They must match the ePO version.

Update Extensions

Check in updated product extensions compatible with new ePO version.

Upgrade Agents

Deploy updated McAfee Agent to endpoints via deployment task or automatic update.
