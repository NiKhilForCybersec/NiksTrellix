Troubleshoot ePO Issues

Common ePO server problems and resolution steps.

ePO

Console Issues

Symptoms:
  • Blank white page when accessing console
  • Browser timeout errors
  • HTTP 500 errors
Resolution Steps:
  1. Verify ePO services are running:
    Check Services
    Get-Service -Name "McAfee*" | Select Name, Status
# Key services: McAfee ePolicy Orchestrator Server, McAfee ePolicy Orchestrator Application Server
  2. Check port availability:
    Port Check
    netstat -an | findstr "8443"
netstat -an | findstr "443"
  3. Review server.log for errors:
    Log Location
    C:\Program Files (x86)\McAfee\ePolicy Orchestrator\Server\Logs\server.log
  4. Clear browser cache and try incognito mode
  5. Restart ePO services if needed
Resolution Steps:
  1. Verify account not locked in ePO (check orion.log)
  2. If using AD authentication, test AD connectivity
  3. Reset admin password via command line:
    Password Reset
    cd "C:\Program Files (x86)\McAfee\ePolicy Orchestrator\Server\bin"
resetpass.cmd admin NewPassword123!
  4. Check certificate validity for SSO/SAML
Resolution Steps:
  1. Check server resource usage (CPU, RAM, disk I/O)
  2. Review database size and performance
  3. Purge old events:
    Event Purge
    # Run Server Task: Purge Threat Events
# Recommended retention: 90 days
# Check EPOEvents table size in SQL
  4. Index database tables
  5. Increase Java heap if needed (server.ini)

Database Issues

Resolution Steps:
  1. Verify SQL Server is running
  2. Test connectivity from ePO server:
    Test SQL Connection
    sqlcmd -S sqlserver\instance -d ePO_SERVERNAME -U sa -P password
# If fails, check firewall (TCP 1433)
  3. Verify database credentials in db.properties
  4. Check SQL Server logs for authentication failures
Resolution Steps:
  1. Check database and log file sizes
  2. Shrink transaction logs (after backup)
  3. Purge old data (events, client tasks history)
  4. Add disk space or move database files
Check DB Size
USE ePO_SERVERNAME;
EXEC sp_spaceused;

-- Find largest tables
SELECT TOP 10 
    t.NAME AS TableName,
    SUM(a.total_pages) * 8 / 1024 AS TotalSpaceMB
FROM sys.tables t
INNER JOIN sys.indexes i ON t.OBJECT_ID = i.object_id
INNER JOIN sys.partitions p ON i.object_id = p.OBJECT_ID
INNER JOIN sys.allocation_units a ON p.partition_id = a.container_id
GROUP BY t.Name
ORDER BY TotalSpaceMB DESC;

Repository Issues

Resolution Steps:
  1. Verify internet connectivity from ePO server
  2. Check proxy settings in ePO
  3. Verify Trellix update sites are accessible:
    • update.nai.com
    • download.mcafee.com
  4. Check firewall rules (HTTPS outbound)
  5. Review server task log for specific errors
Resolution Steps:
  1. Verify distributed repository credentials
  2. Test network connectivity to repository servers
  3. Check disk space on destination repositories
  4. Verify share permissions (UNC path access)
  5. Review replication task configuration

Key Log Files

Log File Purpose
server.log Main ePO server operations
orion.log Authentication, user sessions
console.log Console/UI issues
tomcat-stdout.log Java/Tomcat errors
server-task.log Scheduled task execution
Log Location
