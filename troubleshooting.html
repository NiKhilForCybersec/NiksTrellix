Common ePO Issues

Symptoms:
  • Browser shows connection error
  • Timeout when accessing ePO console
  • HTTP 500 or 503 errors
Troubleshooting Steps:
  1. Verify ePO services are running:
    PowerShell 
    Get-Service | Where-Object {$_.Name -like "*McAfee*" -or $_.Name -like "*ePO*"}
  2. Check port availability:
    PowerShell 
    Test-NetConnection -ComputerName localhost -Port 8443
Test-NetConnection -ComputerName localhost -Port 443
  3. Review server.log for errors
  4. Restart ePO services:
    Command 
    net stop "McAfee ePolicy Orchestrator Server"
net start "McAfee ePolicy Orchestrator Server"
Symptoms:
  • "Unable to connect to database" errors
  • Slow console performance
  • Event processing delays
Troubleshooting Steps:
  1. Verify SQL Server connectivity:
    Command 
    sqlcmd -S SQLSERVER -d ePO_SERVERNAME -Q "SELECT 1"
  2. Check SQL Server service status
  3. Verify SQL authentication credentials
  4. Check database size and transaction log:
    SQL Query 
    SELECT name, size * 8 / 1024 AS 'Size_MB'
FROM sys.database_files;
  5. Run ePO database health check query
Symptoms:
  • Distributed repositories out of sync
  • Agents failing to get updates
  • Replication tasks failing in ePO
Troubleshooting Steps:
  1. Verify network connectivity to distributed repositories
  2. Check repository credentials
  3. Verify disk space on repository servers
  4. Check firewall rules for repository ports
  5. Review Server Task log for replication errors
  6. Manually trigger repository pull task

Common Agent Issues

Symptoms:
  • System shows as "Not Managed" or stale in ePO
  • Last communication time is outdated
  • Agent icon shows red X
Troubleshooting Steps:
  1. Verify agent services are running:
    Command 
    sc query mfemms
sc query masvc
  2. Force agent check-in:
    Command (Run as Admin) 
    cd "C:\Program Files\McAfee\Agent"
cmdagent.exe -p
cmdagent.exe -c
  3. Verify ePO server/handler connectivity:
    Command 
    telnet epo-server.company.com 443
telnet agent-handler.company.com 443
  4. Check sitelist.xml for correct server entries
  5. Review masvc.log for connection errors

Quick Agent Status

Run cmdagent.exe -i to display current agent status, server list, and last check-in time.

Symptoms:
  • Policy changes not applying to endpoints
  • Endpoint shows old policy version
  • Settings don't match ePO configuration
Troubleshooting Steps:
  1. Verify agent is communicating (check last comms time)
  2. Force policy enforcement:
    Command 
    cmdagent.exe -e
  3. Wake up agent from ePO console
  4. Check System Tree location and policy assignment
  5. Verify policy inheritance and overrides
  6. Review agent.log for policy errors
Common Causes:
  • Existing security software conflicts
  • Insufficient permissions (admin rights required)
  • Windows Installer issues
  • Framework requirements not met (.NET, VC++)
Resolution Steps:
  1. Remove competing AV software
  2. Verify local admin rights
  3. Check Windows Installer service
  4. Use verbose logging:
    Command 
    FramePkg.exe /install=agent /silent /log=c:\temp\agent_install.log
  5. Review installation log for specific errors

Common ENS Issues

Symptoms:
  • High CPU usage by mfetp.exe
  • Slow application launches
  • File operations taking too long
Resolution Steps:
  1. Review exclusions ‚Äî add legitimate high-I/O paths
  2. Check scan settings (disable unnecessary scan types)
  3. Reduce archive scan depth
  4. Enable scan caching for clean files
  5. Verify DAT files are current (outdated DATs can be slower)
  6. Analyze with Process Monitor to identify specific bottlenecks

Symptoms:
  • DAT version outdated by multiple days
  • Update tasks showing failures
  • Manual updates failing
Resolution Steps:
  1. Verify repository connectivity:
    Command 
    cmdagent.exe -r
  2. Check sitelist.xml for correct repository entries
  3. Verify repository has current content
  4. Force update from specific repository
  5. Check proxy settings if applicable
  6. Review maupdater.log for errors
Resolution Steps:
  1. Identify the specific detection in Threat Prevention log
  2. Check detection name and file hash
  3. Verify if legitimate application via:
    • Known vendor signature
    • VirusTotal analysis
    • GTI reputation lookup
  4. Create appropriate exclusion:
    • By file path (least preferred)
    • By file hash (SHA-256 preferred)
    • By certificate (for signed software)
  5. Test exclusion in limited scope before deployment
  6. Report false positive to Trellix Labs

MER Collection Guide

The Minimum Escalation Requirements (MER) tool collects diagnostic data required for Trellix support cases. Always collect MER before opening a support ticket.

Running MER Tool

Download MER Tool
Download the latest MER tool from Trellix support portal or extract from ePO console (Menu ‚Üí Software ‚Üí MER).
Run as Administrator
Execute MER.exe with administrative privileges. Select relevant products to collect logs for.
Wait for Collection
MER collects logs, configuration, and system information. This may take 5-15 minutes depending on log sizes.
Upload Results
Locate the generated .tgz file and upload to your Trellix support case.
Command-Line MER Collection 
# Silent MER collection with all products
MER.exe -silent -all

# MER for specific products
MER.exe -silent -products "MA,ENS,DLP"

# MER with custom output location
MER.exe -silent -all -output "C:\Temp\MER_Output"
