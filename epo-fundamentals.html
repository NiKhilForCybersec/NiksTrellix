What is ePolicy Orchestrator?

Trellix ePolicy Orchestrator (ePO) is the centralized security management platform that serves as the command center for all Trellix security products. Originally developed by McAfee, ePO has evolved over two decades to become one of the most widely deployed enterprise security management platforms globally.

ePO is not a security product itself â€” it's the management layer that orchestrates all other Trellix products. Think of it as the "brain" that coordinates ENS, EDR, DLP, and other security modules across your enterprise.

Core Components

Understanding ePO's architecture requires knowledge of its core components. Each plays a specific role in the management infrastructure.

The central application server that provides:

  • Web Console â€” Browser-based management interface (Apache Tomcat)
  • ePO Services â€” Core application services and event processing
  • Event Parser â€” Processes incoming events from agents
  • Built-in Agent Handler â€” Initial agent communication endpoint
Default Ports 
Web Console:    TCP 8443 (HTTPS)
Agent Handler:  TCP 443 (HTTPS) or TCP 80 (HTTP - legacy)
Wake-up:        UDP 8081
Agent Broadcast: UDP 8082

The backend database storing all ePO data:

  • System Tree â€” Organizational hierarchy of managed systems
  • Policy Catalog â€” All policy definitions and assignments
  • Events & Alerts â€” Threat events, audit logs, and alerts
  • Software Repository â€” Package metadata (binaries stored separately)
  • Query Results â€” Report data and dashboard content

Database Sizing

Database size grows significantly with event retention. Plan for 1-2GB per 1,000 managed endpoints with 90-day event retention.

Dedicated servers that handle agent communication at scale:

  • Load Distribution â€” Each handler supports ~10,000 agents
  • Geographic Distribution â€” Place handlers near endpoints for performance
  • Failover â€” Agents automatically fail over between handlers
  • Event Aggregation â€” Handlers buffer events before forwarding to ePO

When to Deploy Agent Handlers:

  • More than 10,000 managed endpoints
  • Multiple geographic locations
  • DMZ or isolated network segments
  • High-availability requirements

Software distribution points for agents to retrieve updates:

  • Master Repository â€” Primary source on ePO server, receives updates from Trellix
  • Distributed Repositories â€” HTTP/UNC shares for local distribution
  • Super Agent Repositories â€” Peer-to-peer distribution via designated agents
Repository Type Protocol Use Case
Master HTTP/HTTPS Primary source, ePO server built-in
HTTP Distributed HTTP/HTTPS Branch offices, cross-firewall
UNC Distributed SMB/CIFS Windows file shares, internal LAN
FTP Distributed FTP Legacy, rarely used
Super Agent HTTP Remote sites, peer-to-peer

Designated agents with enhanced capabilities:

  • Local Repository â€” Cache updates for nearby agents
  • Wake-up Relay â€” Forward wake-up calls to local subnet
  • Bandwidth Optimization â€” Reduce WAN traffic for remote sites

Super Agent Best Practices:

  • One Super Agent per subnet/VLAN
  • Use dedicated, always-on systems (servers preferred)
  • Configure lazy caching to minimize storage
  • Enable wake-up relay for remote management

Upgrade Recommendation

Organizations running ePO 5.9.x or earlier should plan immediate upgrades. These versions are end-of-life and no longer receive security patches.

Ports & Protocols Reference

Proper firewall configuration is essential for ePO functionality. Below are all ports used by ePO and its components.

Port Protocol Direction Purpose
TCP 8443 HTTPS Inbound ePO Console (Web UI)
TCP 443 HTTPS Inbound Agent-to-Server Communication
TCP 80 HTTP Inbound Agent Communication (Legacy)
UDP 8081 UDP Outbound Agent Wake-up Calls
UDP 8082 UDP Broadcast Super Agent Broadcast
TCP 1433 TDS Outbound SQL Server Connection
TCP 8444 HTTPS Inbound Agent Handler Console
TCP 8801 HTTPS Inbound TIE Server Communication
Windows Firewall Rules (PowerShell) 
# ePO Console
New-NetFirewallRule -DisplayName "ePO Console" -Direction Inbound -Protocol TCP -LocalPort 8443 -Action Allow

# Agent Communication  
New-NetFirewallRule -DisplayName "ePO Agent HTTPS" -Direction Inbound -Protocol TCP -LocalPort 443 -Action Allow
New-NetFirewallRule -DisplayName "ePO Agent HTTP" -Direction Inbound -Protocol TCP -LocalPort 80 -Action Allow

# Agent Wake-up
New-NetFirewallRule -DisplayName "ePO Wake-up" -Direction Outbound -Protocol UDP -LocalPort 8081 -Action Allow

# Super Agent Broadcast
New-NetFirewallRule -DisplayName "ePO SA Broadcast" -Direction Inbound -Protocol UDP -LocalPort 8082 -Action Allow

McAfee Agent (MA) / Trellix Agent

The McAfee Agent (now Trellix Agent) is the foundation for all endpoint management. It provides the communication layer between endpoints and ePO.

Agent Responsibilities

  • Policy Enforcement â€” Receives and applies policies from ePO
  • Event Reporting â€” Forwards security events to ePO server
  • Update Management â€” Downloads and applies product updates
  • Product Orchestration â€” Manages installed security products
  • Self-Protection â€” Prevents tampering with agent and products

Communication Type Default Interval Configurable Purpose

Agent Deployment Methods

Agents can be deployed via ePO push, Group Policy, SCCM/Intune, login scripts, or manual installation. For large deployments, consider using SCCM or Intune for staged rollouts with reporting.
